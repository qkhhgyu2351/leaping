## 第一章：选题规则

> **MVP产品详见：** [03财经MVP商业化框架.md](./03财经MVP商业化框架.md)
> 内容配比围绕MVP设计，但不是每篇都带转化钩子。先通过解决目标人群关心的普遍性痛点，赚流量、赢信任，成交MVP是信任积累后的自然收网。

### 1.1 选题三问（发内容前必过）

1. 标题里有**具体工具名或具体数字**吗？
2. 受众看完知道**具体能带走什么**吗？
3. 有没有**"我怎么看"**的视角？

三问都yes，基本方向没问题。

### 1.2 内容红线

| 可以做 | 不能做 |
|--------|--------|
| 解决具体痛点的内容 | 泛泛而谈的内容 |
| 有实操价值的内容 | 纯焦虑情绪的内容 |
| 建立专业信任的内容 | 蹭热点的内容 |
| 符合目标人群画像的内容 | 和定位偏离的内容 |
| 有爆款验证的方向 | 播放量低且涨粉差的方向 |

### 1.3 选题方向

**✅ 能做**

- 税务合规小切口（真实案例、具体罚单、血淋淋的案发现场）
- CFO/投资人视角的判断（专业祛魅、反共识泼冷水）
- 具体企业工具测评（带算账：能省多少、风险值多少）
- AI在财务场景里能做什么、不能做什么

**❌ 不能做**

- 新闻热点搬运（播高粉低，不转化）
- 百分比堆砌的AI焦虑文
- 居高临下的教科书式说教
- 宏观经济评论
- 通用AI工具使用教学（无差异化）
- 已经被同行洗稿无数遍的陈词滥调

### 1.4 平台策略

| 平台 | 策略 |
|------|------|
| **小红书** | **主阵地**，税务合规选题已验证，可以系列化 |
| **抖音** | CFO/投资人IP驱动视角，不追热点 |
| **视频号** | 维持现状，不追加投入 |

### 1.5 内容黄金配比

| 配比 | 内容方向 |
|------|----------|
| 50% 信任/专业 | AI诊断逻辑、合规风险分析 |
| 30% 流量/共情 | 企业乱花钱的坑 |
| 20% MVP/硬广 | 顺带提咨询或报告 |

> 核心理念：如果所有内容都只围着MVP转，那就不是在经营"门店"，而是在发"传单"。内容是"信任的货币"，先通过解决目标人群关心的普遍性痛点（赚流量、赢信任），成交MVP只是信任积累后的自然收网。

### 1.6 目标人群

写给那些"怕在AI浪潮中掉队、怕被审计穿透、怕ROI为负"的30-45岁财务主管、企业主看。

## 第二章：写作规则

### 2.1 语言红线：禁用词/句

**禁用营销套话**
"那么问题来了"、"震惊"、"错过等一年"、"家人们谁懂啊"。

**禁用低智反问**
"难道不是吗？"、"你还在等什么？"等逼迫感反问。

**拒绝泛泛而谈**
"在当前复杂的经济形势下"、"我们要与时俱进"等空话。

**删除情绪废词**
"其实"、"真的"、"非常"、"大概"等虚词。

**禁用正确的废话**
- "真正的高手"——自以为是
- "靠AI思维"——太抽象，像卖课
- "AI不会取代你，但会用AI的人会取代你"——被说烂了
- "这很重要"——定性不说事实
- "大家都在……"——数据不如一个场景
- "说实话"——废话
- "好消息是……"——废话前缀
- "划重点"——假设受众不知道哪个重要
- "你Get到了吗"——居高临下

**禁用搭台子句式**
- "你有没有想过……"
- "其实很简单……"
- "我来告诉你……" → 替换成"我学到一件事"
- "你必须……" → 替换成"我一般会……"
- "你该怎么做" → 替换成"我最近在尝试"

**禁用标签化自己的词**
- "职业病"——在非同行面前显得自high
- 用"养成一个习惯""有个改不掉的习惯"替代

**受众视角自检**
每写一段，自检：目标受众（企业主/CFO）听了会觉得"在说我"还是"关我什么事"？

**好案例公式**
好案例 = 数字具体 + 反常识 + 一句话定性
- 数字具体："28万""4.7年""0.5人"
- 反常识："预期2年回本，实际4.7年"
- 一句话定性："不是AI骗人，是ROI预期从一开始就错了"

**一观点一案例原则（硬性规则）**
- 表达一个观点，必须配一个具体案例
- 一个视频/文章只讲一个主题，可以有分题，但每个分题必须服务于这个主题
- 分题与分题之间逻辑相连，不能各讲各的
- 禁止：一个观点只说结论，没有案例支撑；或一个视频塞进多个不相关主题

### 2.2 文案语气原则

内行说实在话。不是术语堆砌，不是飘在空中讲大词，是真的懂、讲真的。

- 亲近感：不是高高在上的专家，是接地气的内行
- 真的懂：有真东西，不是耍嘴皮子
- 实在：接不接地气，用具体例子和真实数字说话，不空对空
- 能讲宏观：讲落地的方式讲，不飘在空中
- 结构：我见过什么 → 我学到什么 → 你可以带走什么

### 2.3 定量规则

- 时间：具体日期或时间段，不用"最近""目前""当前"
- 数字：百分比/金额/人数具体化，不用"很多""不少""大量"
- 案例：主体+行为+结果三要素，不空洞
- 数据：所有数据必须附原文链接，无链接数据视为噪音
- 所有定性描述必须配一个定量对比或具象场景

错误示例："AI能极大提高报表处理效率。"（泛泛而谈）

正确示例："以前要3个会计凑2天的合并报表，现在大模型10分钟跑完初稿，甚至能顺便标记出5个异常往来款。"（有病灶、有痛点、有结果）

### 2.4 开头框架

**程前框架（内容质感）：** 开头必须是谜题，优先Wow型或Why型。

谜题5类 + 示例：
- How（如何）："如何用3000块做出院线级短剧？"
- Why（凭什么）："一个农村出身的大姐，凭什么做到上市？"
- What（是什么）："什么是真正的财务自由？"
- What If（假设）："如果把一个人关在房间100天会怎样？"
- Wow（震撼/极致）：极致案例"张雪用20年追一辆摩托车——然后造了一个摩托车品牌。"

判断标准：如果观众第一反应是「为什么」，那就容易成功；如果反应是「哦」，那就代表着听听就完事了。

**郑经框架（商业转化）：** 捷径感（让人觉得有解法）+ 痛点具体化（让人觉得你懂他）+ 明确动作（引到具体行动/产品）。

两套框架可以叠加：程前负责抓住人，郑经负责转化人。

### 2.5 专属写作风格

- **视角：** 企业/老板
- **开头钩子：** 可以用真实案例开场，直击老板的真实痛点——怕花冤枉钱、怕团队用不好AI、怕机会错过
- **核心逻辑：** 算账逻辑——所有结论必须落到"钱"上（省了多少、风险值多少、能赚多少）
- **语气：** 像一个干了17年财务和投资的人，在用具体例子跟你讲真的。不是术语堆砌，不是飘在空中，有话直说，不端着
- **结构：** 我见过什么 → 我学到什么 → 你可以带走什么
- **文章署名：** 何斌 | 金融FMBA·高级会计师·董秘·CIA | 多年企业财务与投资分析经验，本文内容仅为专业观点分享，不构成任何投资建议。投资有风险，决策需谨慎。

### 3.3 口播稿模板

**篇幅：** 800字以内（3-4分钟）

**模块数：** 不超过6个（超过6个观众跟不住）

**结构：**
```
开头钩子（前3秒，可以企业案例开头）

模块1-4（每个模块一个核心观点，有具体数字或案例）

结尾CTA

话题标签（#有二财经 #AI多懂企业企业就有多强 + 4个话题标签）
```

**开头钩子设计方法（四步法）：**
1. **锚定误区**——"大多数企业以为AI能直接省人"
2. **揭示真相**——"实际上AI需要有人先把标准写出来"
3. **给出证据**——"一家企业花了28万，预期2年回本，实际4.7年"
4. **具体展开**——"问题出在哪？三个原因"

**口播 vs 文章的核心区别：**
- 口播是"注意力拍卖"——前3秒决定去留，后30秒决定是否看完
- 不能超过6个模块，否则观众跟不住
- 禁止出现运营复盘内容（"没有转化链路""流量不够""没有东西引到产品"——这些是内部讨论，不是对外内容）
- 禁止连续出现"我不知道"等削弱权威的词（整篇不超过1次）
- 禁止叙事主体跳跃——开头是"我"，中段不能突然变成"财务的人"或"企业"，后段再切回"我"
- 每个模块自检："老板为什么要听这段？"
- 提到的数字、文档、案例必须列全（不能说8个只列5个）
- CTA不能太晚出现（不能1800字才到CTA）

**拍摄前必检：**
- [ ] 字数不超过800字
- [ ] 模块不超过6个
- [ ] 前3秒有钩子（锚定误区→揭示真相→给出证据→具体展开），或考虑程前框架
- [ ] 视角从头到尾一致（企业/老板视角），没有跳跃
